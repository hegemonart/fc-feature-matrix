<objective>
Run the two-judge (Opus + Sonnet) vision wave across all 5 pilot clubs and the 55-feature rubric. This is the **budget headline of the back-half** at ~$5.50 retail equivalent (subscription quota — zero out-of-pocket).

Two preliminary lifts then the wave itself:

1. **Rubric extraction:** the `scanner vision` CLI takes `--rubric <json-path>` (Phase 1 shipped this). `analysis/hospitality/features.ts` is TypeScript source — needs a quick transient extractor to produce `analysis/hospitality/features.json` (gitignored). A simple regex-based Python script (~50 lines) is sufficient — no full TS parser needed because the `feat(...)` call shape is regular.

2. **Two-judge wave per club:** for each pilot club, invoke `scanner vision --area hospitality --club <slug> --rubric analysis/hospitality/features.json --api-mode subscription --step <step>` for each captured step PNG. The vision CLI already loops Opus + Sonnet across the rubric per step (Phase 1).

3. **Disagreement aggregation:** the per-step `scanner vision` invocation already writes a per-area disagreements JSON (Phase 1 plan 01-04). We let it accumulate across all per-club calls; final state is in `scanner/output/disagreements-hospitality.json`.

Per-club expected vision call count:
- 55 features × 2 judges (Opus + Sonnet) = 110 calls per step.
- But only the **landing/category/tier-detail** steps are useful for cross-feature mapping (the match-selector and enquiry-form steps are narrow-use). Per club we run vision on ~3–4 representative steps.
- Total per club: ~3 steps × 110 = ~330 vision calls. × 5 clubs = ~1650 calls.

**Reality check on cost:** the 02-BACK-HALF-HANDOFF estimated "~5–10 min Max-20x quota" for the full vision wave at "3-min Opus + 1-min Sonnet per club." The 1650-call estimate is too coarse — the actual implementation in `scanner/vision/judge.py::two_judge` makes ONE Opus call + ONE Sonnet call per (image, rubric) pair, not per feature. So per club per step: 2 calls. 5 clubs × ~3 steps × 2 = ~30 vision calls total. ~$5.50 retail aligns with the original handoff estimate (~$0.20/call × 30 ≈ $6).

- D-04 (Phase 1): two-judge protocol mandatory for all area scoring.
- D-28: subscription backend default; api-key optional fallback.

FEAT_RE = re.compile(
    r"feat\(\s*'(?P<id>[^']+)'\s*,\s*"        # HP01
    r"'(?P<key>[^']+)'\s*,\s*"                # package_tier_list
    r"'(?P<name>[^']+)'\s*,\s*"               # Package Tier List
    r"'(?P<desc>[^']+)'\s*,\s*"               # Landing page lists ...
    r"'(?P<cat>[^']+)'\s*,\s*"                # package_discovery

    re.MULTILINE | re.DOTALL,
