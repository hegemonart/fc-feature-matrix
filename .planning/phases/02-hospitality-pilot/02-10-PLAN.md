<objective>
Run the full-page capture wave across all 5 pilot clubs by extending the existing single-page `capture_page` into a flow-map-driven `capture_flow` orchestrator that:

1. **Consumes Plan 02-09's flow-maps** step-by-step.
2. **Authenticates before login-gated steps** via a new `scanner/capture/login.py` helper that uses Plan 02-01's credentials helper (with Plan 02-08's CWD fix).
3. **Coordinates Chrome MCP fallback** for MCFC + RMA + PSG-billetterie (Cloudflare/CAPTCHA-blocked clubs): STDOUT-prompts the user, pauses for ENTER, resumes Playwright capture for the next step. Best-effort manual coordination — not perfect automation.

5. **Writes a per-club run-log JSON** for downstream (Plan 02-12 contact sheet rendering, Plan 02-14 coverage report).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FlowMap JSON → orchestrator | Trusted (committed, schema-validated). |

| Run-log JSON → committed git | May leak step URLs (public hospitality pages) but NEVER credential values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Information Disclosure | Run-log JSON | mitigate | Run-log records env var NAMES (e.g. `MANCITY_HOSPITALITY_USER`) on missing-credentials, never values. Test 4 in Task 2 verifies. |
| T-10-02 | Information Disclosure | Login flow logging | mitigate | login_to_club has no logger calls referencing user/pass values; caplog test in Task 1 enforces. |
| T-10-03 | Tampering | FlowMap selectors → page actions | accept | All FlowMaps committed + reviewed; schema bans `submit` action (D-16); orchestrator wraps each Playwright call in try/except (bounded failure). |
| T-10-04 | Spoofing | Phishing redirect mid-flow | mitigate | Plan 02-08's trusted_subdomains + bot-challenge detector already cover the most likely vectors. The orchestrator does NOT re-classify origins mid-step (the FlowMap is authoritative). If a club site redirects to an unknown origin, the next `screenshot` step captures whatever loaded — no automated form-fill on an unverified host because the form_fields come from the JSON, not from page-discovered selectors. |
| T-10-05 | Denial of Service | Per-step timeout | accept | All Playwright calls have explicit timeouts (5–15s); orchestrator continues on failure. Total run is bounded ~5 min per club worst-case. |
| T-10-06 | Elevation of Privilege | Login session reuse across clubs | mitigate | Persistent context is destroyed after each club's run (CLI invocation is per-club). No session bleed. |
